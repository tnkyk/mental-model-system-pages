画面の向こうで起きている奇妙さ

返事が数百ミリ秒で流れ込むとき、背後を忘れやすい。あなたの一文はまず細かい記号の列へ割れ、巨大な行列とキャッシュの読み書きを何度もくぐり、次にありそうな続きが一つずつ選ばれて画面に現れる。そこに「賢い主体」は必須ではない。あるのは、観測された系列の次を当てるよう磨かれた関数と、それを運ぶ資源の束である。

それでも現代の仕事は、ナレッジ検索・設計下書き・コード修正・調査要約を、この仕組みに委ねつつある。Software が岩石を計算機械へ、Networking がケーブル上の雑音を共有プロトコルへ降ろしたように、この記事は LLM を 送信ボタンからシリコンまでの必然性の鎖として組み立て直す。流暢さ・説明・自信を、正しさ・因果・権限の代理にしてしまう前に、機械の解像度を一段ずつ上げる。

読み終えるころには、次の日常の謎が一本の地図に載るはずである。なぜ自信ありげな答えが後から誤るか。なぜ Temperature を下げても嘘が消えないか。なぜ「考えた様子」が正しさの証明に見えるか。なぜ長い文脈でも重要箇所を落とすか。なぜ Agent が危険な変更を流暢に提案するか。なぜ絶対語で語りたくなるか。なぜ新製品の発表で地図が消え、RAG 付きでも引用が流暢なまま誤るか。

すでに触れている裂け目

一つ目の場面。社内のナレッジチャットに「このAPIの現行の認証方式は何か」と聞く。答えは滑らかで、自信ありげで、用語も正しい。三日後、実装を見たらそのAPIは既に別方式へ移行していた。画面上の流暢さは、正しさの代理ではなかった。あなたは「嘘をつく主体」に騙されたのではなく、観測系列から次にありそうな続きを出したプログラムのふるまいに、信用を渡しすぎた。

二つ目の場面。モデルが長い手順を「まずは〜を確認し、次に〜を検討し、結論として〜」と書き連ねる。文章は思考の跡に見える。レビュー会で誰かが「手順が見えるから大丈夫」と言い、検証を後回しにした。見える推論文は出力の一形態であり、内部の完全な写しでも証明書でもない。

三つ目の場面。Coding Agentが差分を提案する。「このロックは冗長です。削除します」と説明し、自信を持ってパッチを出す。そのロックは稀なレース条件を防いでいた。Agentは候補を出すのは得意でも、本番への到達範囲と承認の境界は自動では与えられない。

これらは「AIが賢いかどうか」の話ではない。あなたがすでに感じている 流暢さ・説明・自信 と、正しさ・因果・権限 の裂け目である。裂け目を埋めるには、先に機械が何であるかを梯子で上る。判断の型——探索・変換・検証・権限——は、梯子の頂上で回収する。未来の発表語りも、同じ梯子に刺す。

この記事がすること／しないこと

すること

  • 送信からシリコンまでを、前の層が壊れる場所だけを継ぎ目にして連続説明する（tokenizeize → 埋め込み／位置 → Transformer → decoder-only／文脈条件付け → データ形状 → 学習／適応 → サンプリング → Prefill/KV/Decode → GPU／効率応答 → チャット／Tool／外側観測 → 送信一発の合成）
  • 不完全だった系列モデルが何を欠けていたかから、なぜ Attention 並列混合が必然かを鎖で示す
  • Tokenizer / Attention / Sampling / Prefill–Decode / KV成長 / Permission / Evidence の各 Lab（教育用トイ）で bottleneck を感じ、途中で日常の謎を場面として回収する
  • 機械梯子のあとで、検証と権限を外側に置く判断地図と、使う人／作る人の読み方を畳む

しないこと

  • 価格・投資・採用ROI・どの製品が一位かの予言
  • プロンプト技のカタログや、閉鎖 frontier の非公開回路を事実扱いすること
  • 意識・道徳的地位の結論
  • 製品RAG／Agent の完全仕様カタログ（外側観測の必要性まで；設計の細部は別Issue）

読み終えたとき、あなたは「このモデルは信じられるか」ではなく、「自分は今、どの層に立ち、何を機械に任せ、何を外側に残しているか」を、使う側／作る側のどちらの言葉でも説明できるはずである。

なぜ「保証なき候補生成」が工学的に選ばれるか

裂け目を感じた瞬間、人は「正しいことだけを出力する機械」を欲しがる。だが世界の真偽を直接最適化する損失は、インターネット規模のテキストにはなかなか載せられない。ラベル付きの「この文は真／偽」を全てに貼るのは不可能に近く、仮に貼れてもラベル自体が古く、分野依存で、敵対的になる。

代わりに工学が選んだのは、すでに溢れている観測——人間が書いたテキストの続き——を当てる目的である。次に来るトークンの条件付き確率を高くする。すると機械は、文法・スタイル・よくある事実パターン・議論の型を大量に圧縮できる。その代償として、出力は 尤もらしい候補 であり、真理関数の値ではない。(*2)(*3)

ここが統計と真理の混線点である。「コーパスでよく共起する」は「介入しても成り立つ因果」でも「今日の社内APIの真実」でもない。最初の場面——自信ありげな古い認証方式——は、主体の悪意ではなく、この目的関数の success metric が信用ではなかったことに由来する。

次の限界はこれだ。観測系列を当てるとしても、系列をどう表現し、長い依存をどう扱うかは別問題である。局所的な統計だけでは足りなかった。

不完全だった系列モデル：局所統計と逐次が欠けていたもの

言語の次を当てる試みは、Transformer 以前からあった。古典的な n-gram は、直前の数トークンのカウントで次を予測する。短い窓では驚くほど動く。だが窓の外の束縛（「さっき述べた認証方式」や「この関数の引数」）は原理的に見えない。窓を広げると組み合わせ爆発が待ち受け、希少な長い依存は統計的に死ぬ。

再帰型ネットワーク（RNN / LSTM 系）は、隠れ状態を時間方向に更新して長い文脈を運ぼうとした。理論上は無限の過去を圧縮できる。実践では、勾配が遠い過去へ届きにくく、各ステップが前ステップに依存するため学習時の並列化も難しかった。長い依存と大規模並列学習の両方で、壁が見えた。

Attention Is All You Need が示したのは、再帰や畳み込みに頼らず、系列内の位置同士を内容に応じて直接混ぜられることである。(*1) 学習時には多くの位置をまとめて計算でき、長距離の依存も「何ステップ経ったか」ではなく「関連する位置へのアクセス」として書ける。これが次の層——離散系列上の巨大な混合／変換機械——への必然である。

歴史は博物館展示ではない。n-gram が欠けていたのは視野、RNN が苦しんだのは遠い信号と並列性。その欠落が、のちのブロック設計を要求した。

文字を離散化する：トークンという学習空間

連続する文字をそのまま実数の海で扱う代わりに、現代の LLM はテキストを 語彙上の離散 ID 列 に写す。SentencePiece のようなサブワード方式は、生コーパスから分割規則を学び、語彙にないレアな表記も断片へ割れる。(*2) モデルが見るのは「意味の原子としての単語」ではなく、この規則が定義した記号列である。

同じ文字列でも分割が変われば、系列長が変わり、境界に乗る情報が変わり、損失の単位も変わる。「JWT」が一トークンか三トークンかは、見た目の単語ではなく学習された断片化の問題である。長いプロンプトが「高い」と感じるとき、あなたはしばしば文字数ではなくトークン数と Prefill コストを支払っている。

学習の主目的は、文脈トークン列 \(x_{<t}\) が与えられたとき次トークン \(x_t\) の負の対数尤度——クロスエントロピー——を下げることである。成功は「正しい世界」ではなく「訓練分布上で尤もらしい続き」として定義される。だから最初の場面の古いAPI回答は、目的関数が信用を最大化していなかったことの症状として読める（P1）。

ベクトル空間に載せ、語順を注意へ焼き込む

トークン ID は整数のラベルにすぎない。それを固定長の実数ベクトル（埋め込み）へ引き出すことで、似た文脈で共起する記号が近い方向を持ちうる幾何が生まれる。埋め込み行列は巨大なルックアップテーブルであり、学習で動く重みの一部である。

しかし袋に入れたベクトルの集合だけでは語順が消える。「犬が人を噛む」と「人が犬を噛む」を区別できない。初期の Transformer は三角関数の位置符号化を加算した。(*1) 現代の多くのデコーダでは、絶対位置を Query/Key への回転として自己注意の内積に載せる RoPE 系が使われる。(*6) 要点は実装の流行語ではなく、位置情報が注意のスコア計算に入らなければ順序が機械に見えないという必然である。

ここで解像度が一段上がる。チャットの文字列は、すでに (1) 分割され (2) ベクトルになり (3) 位置の効いた表現として、次の混合層に入る準備ができた。

文脈を混ぜ、特徴を変換する：Transformer ブロック

位置付き埋め込みだけでは、まだ「どの文脈が今の続きに効くか」を解けない。各層はおおむね二つの仕事を繰り返す。(*1)

Attention（混合）を式の感触まで下ろす。 位置 \(i\) の表現から線形写像で Query \(q_i\) を作り、過去位置 \(j \le i\)（因果マスク下）の Key \(k_j\) と内積を取る。スコアを \(\sqrt{d}\) などでスケールし Softmax すると、合計1の重みになる。その重みで Value \(v_j\) を加重和したものが、位置 \(i\) の新しい混合表現である。RoPE が入っていれば、この内積自体が相対位置の関数でもある。(*6) ここまでが「検索エンジン」ではなく いまのウィンドウ内の加重読み出し である理由だ。外部コーパスへ HTTP しているわけでも、行を SELECT しているわけでもない。

Multi-Head は、同じ系列を複数の射影で並行混合する。あるヘッドは構文的な照応に寄り、別のヘッドは数値やコード識別子に寄る、といった分業が学習されることもあるが、ヘッド＝人間の認知モジュールと同一視しない。

MLP / Feed-Forward（変換） は、Attention のあとに位置ごとへ入る。典型的には拡大線形→非線形→戻す線形で、その位置の特徴を組み替える。Attention が「どこを見るか」なら、MLP は「見たものをどの方向へ曲げるか」に近い。層を積み重ねるたびに、局所的な混合と局所的な変換が交互に深くなる。

残差接続と正規化 が、この積み上げを学習可能にする。出力を「前の表現＋差分」として書くことで、深い層でも信号と勾配の通り道が残る。積み上げの頂点近くで、LM Head が logits——次トークン候補の生スコア——を出す。ここまでが、同じ型のブロックを何十層も重ねた塔である。

因果マスクの約束を再掲する。位置 \(i\) は未来を見ない。チャットで返答が左から右へ伸びる体験は、このマスクと自己回帰の公開面である。

長い文脈をプロンプトに詰め込んでも安心できない理由も、ここで芽生える。Attention は均一に全部を「覚えている」のではなく、Softmax 重みの稀少なピークと長い尾である。重要な一文が似た語彙の distractor に負ければ、モデルは「文脈に書いてあるのに見落とす」（P4）。社内ドキュメントをそのまま貼ったのに結論がずれる現象は、まず記憶喪失や陰謀ではなく、混合の失敗として疑う。

デコーダだけの塔と、文脈への条件付け（いわゆる in-context learning）

原論文の Transformer は、翻訳向けに encoder（読む塔）＋ decoder（書く塔） を持っていた。(*1) 多くの現代 LLM チャットは、そのうち decoder-only——因果マスク付きの自己回帰塔——を主柱にする。左から右へ続きを書く機械として訓練し、チャットも「続き」として書く。encoder-decoder が消えたわけではないが、日常で触れる主戦場の第一近似はこちらの向きである。

ここに、使う人が毎日触れている現象が乗る。プロンプトに例を二、三個書くと、追加学習なしで型を真似ることがある。これを in-context learning（ICL）と呼ぶことが多い。機構として先に読むべきは、新しい知性モジュールが生えたことではなく、すでに学習された条件付き分布が、文脈トークンを条件に再パラメータ化されることである。例示・指示・制約は、いずれも文脈ウィンドウへの条件である（P3 / P4）。

「創発」という言葉も、ここで輪郭を与える。スケールやデータで急に見える挙動の変化はある。だがそれを、梯子の外にある別種の主体の誕生と読む必要はない。見えるのは、条件付けと資源と分布が変わった結果のふるまいである。作る人は「どの条件を足したか」を名指しし、使う人は「いまの答えがどの文脈条件に乗っているか」を監査する。

次の限界: この塔の重みは、最初から賢く固定されているわけではない。観測系列の損失を通って、どうやって立ち上るのか。

広大なパターン圧縮：事前学習・勾配・スケール・並列

塔の形が決まっても、中身の数は観測から学ばねばならない。事前学習では、巨大コーパス上で次トークンのクロスエントロピーを下げる。(*2)(*4)

微視例を一分だけ長くする。語彙が極端に小さいトイ世界で、文脈トークン「認証は」のあと、コーパス上の正しい次が「Bearer」だとする。モデルが出した確率が \(p(\text{Bearer}\mid\text{認証は})=0.05\) なら、この一歩の損失は \(-\log 0.05\) と大きい。逆伝播は感度を計算し、オプティマイザが一歩だけパラメータを動かす。一歩は微弱でも、トークン数が巨大になると、文法、慣用句、コードの型、議論の型、誤りの型までが統計として沈殿する。単純なのは保証の形であり、適合の深さではない。

固定の計算予算では、パラメータ数と学習トークン数の配分が効く、というのが Chinchilla 系の示唆である。(*4) 「とにかく巨大にすればよい」は性質の話としては粗い。数値の最適点は時代とデータ混合で動く。脚注は 配分というトレードオフが存在することの足場であり、製品サイズの予言ではない。

さらに、現代規模では 一つの GPU に重み（と活性化、と勾配、とオプティマイザ状態）が載らない。Megatron 系が示したテンソル並列は、層内の大きな行列を複数 GPU に分割し、必要な箇所で集合通信する。(*7) パイプライン並列は層の塔を段階に分ける。(*8) データ並列は複製したモデルが異なるバッチを処理し、勾配を平均する。巨大化はロマンではなく、メモリと通信の工学である。

それでも事前学習が最大化するのは観測尤度である。古いAPIの流暢な誤り（P1）は、圧縮された過去のパターンが、いまの社内真実より強く活性化したときに起きる。学習は「世界の現在」を購読していない。学習は「観測された続き」を圧縮している——ならば次に、何が観測として残ったかを見る。

データの磨き方が、観測分布そのものを作る

同じ「インターネット規模」という言葉の裏で、実際に損失が当たる系列は一様ではない。フィルタ（毒性・低品質・重複の除去）、dedup、品質スコア、ドメイン混合、個人情報や著作権まわりの方針——これらは周辺作業に見えて、実は 尤度の対象分布を設計している。(*4) あなたが感じる「よくある言い回しに引っ張られる」「特定文化の前提が染みている」は、しばしば主体の悪意ではなく、この分布の継承である。

作る人への刺し方は明確だ。データパイプラインを変えることは、目的関数のラベルを変えるのと同じくらい、モデルの性格を変える。使う人への刺し方はこうだ。流暢な誤りを見たとき、「賢いふり」だけでなく「どの観測世界の続きか」を疑う（P1）。バイアスや盲点は、重みの奥に隠れた人格というより、観測分布の影であることが多い。

混同釘: 「もっとデータを積めば真実に近づく」。量は配分の話であり、品質と混合は分布の形の話である。真理関数への自動収束ではない。

次の限界: 生の続き予測分布は、指示に従うアシスタントとしても、安全な同僚としても、まだ形が粗い。振る舞いを別の信号で曲げる工程が要る。

振る舞い分布を動かす：後訓練

指示追従・選好整合・ある種の推論様式は、しばしば別信号で後から形作られる。InstructGPT 系は、デモでの Supervised Fine-Tuning、比較データでの報酬モデル、強化学習による方策更新、というパイプラインを示した。(*3) DPO のように、明示的報酬モデルと RL ループを経由せず選好対から方策を動かす経路もある。(*5) 公開された Reasoning 系は、検証可能な報酬や長い生成を信号に使う例を報告する。(*16)(*17)

重要なのは、後訓練が「真理関数への置換」ではないことだ。変えられるのは生成方策の分布——どんな続きを出しやすいか——である。だから二番目の場面の長い推論文（P3）は、内部の完全な因果計算の写しというより、後訓練と復号が用意した 出力チャネルのスタイル として先に疑う。

混同釘: 「RLHF 済み＝幻覚が消えた」「Thinking モード＝内部で証明した」。どちらも、分布の移動と保証の混同である。

次の限界: フルパラメータ更新だけが適応ではない。小さな差分で方策を動かすレバーがある。

小さなアダプタで動かす：パラメータ効率の適応

すべての用途で全重みを動かす必要はない。LoRA 系が示した原理は、選んだ重み行列に 低ランクの小さな更新 を載せ、基底の事前学習重みは凍らせたまま適応することである。(*19) 学習対象のパラメータは元のごく一部になり、推論時には差分をマージできる、という設計意図が付く。

これを「安い微調整商品」だけに矮小化しない。FP としての読み方はこうだ。すでに圧縮された基底分布に、薄い差分を足す。 別の知能コアではない。使う人は「どのアダプタが今乗っているか」——社内データで足された癖、期限付きの実験タグ——を信頼付着点として監査する。作る人はこのレバーを、データと目的（何に寄せるか）と権限（誰が載せられるか）とセットで設計する。小さくても、悪意データも載る。

次の限界: 分布ができても、実行時にはその分布から 一つの 系列を選ばねばならない。

分布から一つの続きを選ぶ：自己回帰サンプリング

推論の一歩は、logits を Softmax で確率にし、Temperature や Top-p などで形を整え、一つのトークン ID をサンプリング（または greedy に選ぶ）ことである。選ばれたトークンは文脈の末尾に繋がり、同じスタックが繰り返される。誤りも、いったん選ばれれば次の文脈の一部になる。

Temperature を下げると分布は尖り、毎回のばらつきは減る。だが尖る先が「真」である保証はない。訓練・文脈が支持する確からしい誤りへ収束しうる（P2）。低 Temperature はランダム性のノブであって、真理のノブではない。

次の限界: 一トークンずつの一歩は、サービスとしては Prefill と Decode という位相の違う仕事になる。

Prefill と Decode：同じモデル、違う拘束

プロンプト全体を初めて読む位相を Prefill、生成中に一トークンずつ進む位相を Decode と呼ぶことが多い。Prefill では入力系列に対してキー・バリューをまとめて計算し、KV cache に載せられる。Decode では新しいトークンについて計算し、過去の KV を再利用する。(*9)(*10)

同じ「トークン数」でも、位相が違えばレイテンシと資源の効き方が違う。長いシステムプロンプトと添付文書は Prefill を重くし、だらだら長い回答は Decode を伸ばす。ユーザー体験の「考えている間」と「文字が流れる間」は、しばしばこの位相差の感触である。

次の限界: 一人の対話ならまだしも、多数の要求が同時に来ると、KV の置き方が製品を支配し始める。

メモリとバッチ：KV が伸び、断片化が効く

自己回帰生成では、過去トークンの K/V を保持し続ける。出力が伸びるほど、その対話の KV は増える。多数の利用者が同時に長く話すと、GPU メモリは重みだけでなく KV で圧迫される。単純な連続確保は長さのばらつきで断片化し、同時処理数を落とす。PagedAttention 系は、OS のページングに似た非連続管理でこの問題に応える。(*9) 連続バッチは、待ちと生成中の要求をスケジューラが混ぜ、GPU を遊ばせないための必然である。

投機的復号のように、草案モデルで複数トークンを提案し本体で検証する加速もある。(*11) 多くは「分布を壊さず平均レイテンシを下げる」設計を目指す。ここでも性質と製品ベンチの予言を混ぜない。

シリコン上の拘束：行列積と帯域

GPU 上の LLM 実行では、巨大な行列乗算と、重み・活性化・KV のメモリ移動が支配的になりやすい。(*12)「頭の中で考える速さ」ではなく、HBM からどれだけバイトを運び、Tensor Core でどれだけ FLOPs を消化できるかの問題である。だから低精度、カーネル融合、FlashAttention のような IO 意識的実装が効く。(*10)(*12) それはしばしば数学的定義を変えるというより、同じ計算を帯域の現実に落とす工夫である。

訓練時のテンソル／パイプライン並列も、同じメモリ天井への応答であった。(*7) 推論時の KV ページングも同様である。Hardware 節は製品カタログではなく、前の層で見た拘束がどの物理量に翻訳されるかの地図である。

混同釘: 「パラメータ数＝知能」「GPU 数＝正しさ」。どちらも資源の話と真理の話の短絡である。

次の限界: 同じ天井に対して、業界が繰り返し出す「効率の新語」を、拘束への応答として名指しする。

効率への応答：疎な専門家と量子化

発表が「革命」に聞こえるとき、中身はしばしば次のどちらか（または両方）である。拘束を緩める工夫か、近似で支払う工夫か。

Mixture-of-Experts（MoE） は、すべてのトークンに全パラメータを起こす代わりに、ルーティングで一部の専門家だけを動かす疎性の設計として読める。(*16)(*17) 見えるパラメータ総量が大きくても、トークンあたりの起動は小さくなる、というのが狙いの骨格である。これは「別種の脳が生えた」話ではなく、計算予算への攻撃である。

量子化は、重みや活性化のビット幅を落とし、メモリと帯域の請求書を軽くする近似である。(*12) 定義を捨てるというより、同じ計算を粗い数値で運ぶ。品質とのトレードオフは実験の問題であり、「量子化したから賢くなった／馬鹿になった」の一言では足りない。

使う人（P7）: 新語に地図を捨てない。いま緩んだのは計算量か、メモリか、レイテンシか。作る人: どの拘束を攻撃し、どの品質レバーを差し出しているかを名指しする。

次の限界: シリコンの上でトークンが流れるとして、チャットアプリがモデルに見せているものは何か。

アプリ層のプロトコル：テンプレートと Tool

画面上の吹き出しは、モデルの入力そのものではない。メッセージ列は、モデルが学習した書式——special tokens や役割マーカーを含む文字列——へ chat template で写されてからトークン化される。(*13) Networking が「共有テーブル」なしにパケットを解釈できないように、テンプレート不一致は「モデルが壊れた」のではなくプロトコル層の失敗として診断できる。

Tool / function calling では、利用可能な関数のスキーマが文脈に載り、モデルは呼び出しを生成トークンとして提案できる。だが モデル自身はツールを実行しない。ホストが呼び出しを解釈し、実行し、結果をまたテンプレート経由で戻す。(*13) 三番目の場面——ロック削除の自信ある提案——は、ここを権限と読み違えたときに悪化する（P5）。流暢な delete_lock() 提案は、本番実行の承認ではない。

次の限界: Tool 提案の一歩外側に、検索や多段エージェントという「観測ループ」が広がる。製品ツアーではなく、機構として固定する。

外側の観測ループ：retrieval とエージェント

重みの中だけに知識を閉じない、というのが外側観測の動機である。Retrieval-Augmented Generation（RAG）系の骨格はこう読める。外部コーパスから文書を取得し、その内容を 生成の条件（文脈）へ載せる。(*20) モデルは、取得パッセージを見た上で続きを書く。Agent はその兄弟で、観測チャネルをブラウザ・シェル・チケット・社内APIなど行為可能な環境へ広げる。いずれも「ホスト側がループを持つ」契約であり、モデル単体が真実を購読したことにはならない。

ここが P8 の現場である。引用がきれいで、リンクがあり、口調が誠実でも、取得集合が古かった／取り違えた／混合で負けた／サンプリングが尖った、のいずれでも誤りは残る。Permission Gate で感じた分離は、retrieve 結果にもそのまま効く。流暢な citations は証拠梯子の二段目に寄りやすいが、介入検証と承認を飛ばさない。

混同釘: 「RAG したから hallucination は消えた」「Agent だから責任ある」。外側ループは観測と行為のチャネルを増やす。検証と権限はなお外側である。

次の限界: 層を並べただけでは、読者の解像度はまだ棚卸しで終わりうる。Networking が「リンクをクリックすると何が起きるか」で閉じたように、送信一発で鎖を合成する。

送信を押すと何が起きるか

Networking の記事が「リンクをクリックすると何が起きるか」で層を一度に貫通させたように、ここでも導入の場面を 一本の時間順で撃ち抜く。あなたが社内チャットで「このAPIの現行の認証方式は何か」と打ち、送信を押したとする。裏側で起きうる鎖は、およそ次である（実装細部は省略した教育用の一本道。製品固有の最適は主張しない）。

  1. UI → メッセージオブジェクト: 役割（system / user / 過去の assistant）と文字列が構造化される。画面の吹き出しはまだモデル入力ではない。
  2. （任意）外側観測: ホストが検索や社内インデックスを呼び、取得文をメッセージへ差し込むことがある。(*20) ここで入れる観測の質が、のちの尖りを半分決める。
  3. Chat template: モデル固有の special tokens と役割マーカー付きテキストへ線形化される。(*13) テンプレートがズレると「モデルが急に馬鹿になった」ように見え、実体はプロトコル不一致である。
  4. Tokenize: サブワード分割で ID 列になる。境界と長さ（Prefill の課金単位の片方）がここで確定する。(*2) Tokenizer Lab で感じた「同じ文字列・違う長さ」が、いま請求書とレイテンシに効く。
  5. Embed + 位置: ID がベクトルになり、RoPE 等が注意内積に順序を載せる。(*6)
  6. Prefill: 塔を下から上へ通し、各層の Attention が文脈を混ぜ、MLP が変換し、KV が層ごと・位置ごとに積まれる。(*1)(*9) システムプロンプトと添付仕様が長いほど、この位相が重い。アダプタが載っていれば、基底＋差分の合成として塔が動く。(*19)
  7. Logits → 最初のサンプル: LM Head の分布から最初の応答トークンが選ばれる。Temperature / Top-p は尖りと候補集合を変えるだけである（Sampling Lab）。
  8. Decode ループ: 選ばれたトークンが文脈末尾へ繋がり、新しい K/V が伸び（KV Growth Lab）、次トークンが流れる。画面のストリーミングはこのループの可視面である。長い「考えた様子」の文章も、多くの場合このループが長く続いただけの出力チャネルである。
  9. 停止: EOS、長さ上限、停止文字列、あるいは Tool call パターンの検出で止まる。
  10. Tool なら外側へ: 呼び出し文字列はオーケストレータが受ける。実行するかどうかは権限ポリシーである。(*13)(*15) Permission Gate Lab の境界がここに差し込まれる。結果が文脈へ戻り、Decode が続くこともある（外側ループ）。
  11. あなたが読む: 流暢な「OAuth2 の Bearer です」が現れる。それが社内の現行実装と一致するかは、この鎖のどこにも保証として含まれていない。

古くなった認証方式の回答は、ステップ6–8で尖った分布が過去パターンを出した結果でありうる。長い文脈の見落としは、ステップ6の混合の失敗でもありうる。危険なパッチ提案は、ステップ10で権限を飛ばした結果でありうる。引用付きの誤りは、ステップ2の取得品質とステップ6の混合の合成でありうる（P8）。

検証は、リポジトリを開く・ステージングで試す・所有者に聞く・テストを走らせる、という 別の鎖 である。送信の鎖と、真実の鎖と、権限の鎖を、ここで分離できる。これができると、次の節の「四役割」はスローガンではなく、いま貫通した層の要約になる。

探索・変換・検証・権限：梯子の上の判断地図

機械梯子が見えた今こそ、役割を畳める。

  • 探索（重み内）: 圧縮パターンの活性化
  • 探索（文脈内）: Attention による位置選択（ICL の条件も含む）
  • 探索（外部）: Tool／retrieve／実行環境
  • 変換: 表現を別表現へ写す（層と Head の仕事）
  • 検証: テスト・出典・再現・人間レビュー
  • 権限: 誰がどの操作を実行してよいか

有用なシステムは、探索と変換を機械に任せすぎる一方で、検証と権限を省略すると壊れる。比喩の有効域を越えないこと。Attention を「社内検索の代替」と呼ぶと、出典と権限が消える。RAG を「検証の代替」と呼んでも同じ欠落が起きる。

因果の限界：語ることと同定することは別

観測系列上の関連（association）と、介入したときの効果（intervention）と、反実仮想の同定は層が違う。(*14) 「原因はこれです」と自然に書けることと、介入して確かめられることは別である。P6 の絶対語——改ざん不可、確実、証明済み——は、観測尤度機械に因果の勲章を着せたい誘惑である。

二番目の場面の長い推論文に戻る。過程が尤もらしく見えることと、反実仮想が同定されたことは違う。因果の主張は、Test・実験・監査ログなどの介入チャネルへ送る。

実践：証拠梯子と権限

候補の証拠強度を、おおよそ次のように上げる。(*15)

  1. 未検証のモデル出力（ただの候補）
  2. 出典付きだが未確認（リンクや引用がある——RAG の返りも含む）
  3. 介入検証（テスト実行、再現、一次データ照合）
  4. 権限付き承認（破壊的操作・外部発信・支払いなどの到達範囲）

Coding Agent のロック削除提案（P5）は、1 から 4 を飛ばしたときに事故る。引用付き誤り（P8）は、2 で止まったときに事故る。Permission Gate Lab で感じた分離を、運用ルールに落とす。

マルチモーダル：整列された表現の境界

画像や音声を「読む」製品は、しばしば別主体に見える。機構の第一近似はこうだ。他モダリティは、エンコードや整列によって 系列機械が読める表現の境界 を越えて入る。(*1) いったんトークンや埋め込みの列として塔に入れば、仕事はなお混合／変換と次の候補の選択である。

具体的には、画像はパッチや検出器を通ってベクトル列になり、言語側の埋め込み空間と揃う（対照学習・投影・共有トークンなど経路は複数ある）。揃ったあとで起きることは、テキストだけのときと同じ型——どの位置を混ぜ、どう曲げ、次の候補を出すか——である。新モダリティは、入力契約と表現境界を足すことが多く、「思考主体が目覚めた」ことと同義ではない（P7）。

使う人: 画像付きの流暢な答えも、検証と権限の外には出ない。写真の誤読・図の取り違えも候補生成として起きうる。作る人: どの境界で表現を揃え、どの損失で引っ張っているかを名指しする。境界の設計変更を「別知能」として売らない。

二つのプレイブックと、技術革新の刺し方

ここが主要クロージング判断である。梯子を登ったあとに畳むのは、「賢いか」ではなく 明日の発表をどこに刺すか である。

使う人のプレイブック：信頼の付着点を監査する

いまの答えは、次のどの付着点に乗っているか。

付着点監査の問い
重みどの基底／どのアダプタ／どの後訓練の癖か
文脈どの指示・例示・添付が条件になっているか（ICL）
外部観測retrieve／Tool の結果は何か、古くないか、取り違えていないか
権限提案と実行の境界はどこか、誰が承認するか

流暢さ・長い推論文・きれいな引用は、いずれも付着点の 装飾 になりうる。装飾で証拠梯子を飛ばさない。

作る人のプレイブック：どこを攻撃しているかを名指しする

設計変更や研究発表は、だいたい次のどこかを攻める。

攻撃点問い

目的CE／SFT／選好／検証報酬のどれか
建築decoder-only／MoE／位置／マルチモーダル境界のどれか
推論Prefill/Decode、KV、test-time 探索、サンプリングのどれか
外側契約template、retrieve、Tool、権限、人間承認のどれを足したか

「革命」語を聞いたら、まずこの表に落とす（P7）。製品名の言い換えに地図を捨てない。

よくある誤解

「次トークン予測だけだから単純である」

単純な目的が、巨大な混合／変換と資源工学と後訓練を排除しない。単純なのは保証の形であり、実装の浅さではない。

「探索と変換だけでは新しいものを作れない」

候補空間の組合せは実質的に膨大である。新しいように見える出力は出る。問題は、新しさと正しさと許可が同じでないことだ。

「Chain of Thought は内部思考の完全な記録である」

見える推論文は出力チャネルである。内部活性化の完全転写でも、因果の証明書でもない。

「in-context learning や創発は別種の主体の証明である」

ICL は文脈への条件付けとして先に読む。スケールで見える挙動変化も、梯子の外の主体誕生と同一視しない。(*1)

「Tool や RAG を使えば hallucination はなくなる」

外側観測は観測を増やす。誤取得、誤読、混合の失敗、権限超過は残る。(*20)

「MoE や量子化は新しい知能の形である」

多くは計算・メモリ拘束への応答である。疎性と近似を、真理保証と混同しない。(*16)(*12)

「マルチモーダルになったから別物の意識がある」

多くは整列された表現境界の拡張である。検証と権限はなお外側である。

「bigger なら smarter」

パラメータ増加は資源とデータの配分の話であり、正しさ・因果・権限の単調関数ではない。(*4)

「low Temperature なら正しい」

尖る先が真とは限らない。

「long context なら全て覚えている」

Attention は寄与の混合である。長さは焦点の保証ではない。

「benchmark が高いから仕事を自律化できる」

ベンチは特定分布上のスコアである。権限と証拠梯子の代替ではない。

「causal explanation が自然なら cause を理解している」

語りと介入同定は別である。

「学習済み／推論済みだから改ざん不可」のような絶対語

観測尤度機械に、監査可能性や最終性の勲章を着せない。

反例・限界

  • 閉鎖モデルの内部は、公開報告の範囲を超えて断言しない。
  • 教育用 Labs は生産メーターではない。
  • 研究前沿（MoE、test-time compute、解釈可能性）は地図であり、単一の AGI 核心の発見物語ではない。(*16)(*17)(*18)
  • この記事自体も、特定製品の安全性証明にはならない。

自分は今、地図のどこに立っているか

次の問いに、自分の仕事の言葉で答えられれば、この記事の梯子は手元に残っている。

使う人向け

  1. いまの出力は、重み／文脈／外部観測／権限のどの付着点に乗っているか？
  2. 証拠梯子は何段目まで登ったか（未検証／出典／介入／承認）？
  3. 破壊的操作の権限境界は文書化されているか？
  4. 停止条件と人間の責任は先に決めたか？

作る人向け

  1. いまの変更はデータ・目的・建築・推論・外側契約のどこを攻撃しているか？
  2. 緩めた拘束と、足した契約は何か（innovation plug-map）？
  3. 失敗時にユーザーが触る付着点はどこか、それは監査可能か？
  4. 成功条件・停止条件・予算は先に決めたか？

例（使う人）: 「自分は編集者として、変換（下書き）だけを機械に任せ、出典照合と公開権限は外に残す」。例（作る人）: 「自分はプラットフォームとして、retrieve と Tool をサンドボックスに閉じ、生産認可を別キューにする」。(*15)

立ち位置が言えたとき、流暢さは便利な UI であって、世界の主人ではなくなる。新発表も、配線図の上の一点として落ちてくる。

研究地図（短いコダ）

進歩は単一の核心の開示というより、複数制約への攻撃の束として読むほうが実務に近い。混合の効率（MoE 等）、推論時計算、メモリ／サービング、解釈可能性の介入可能性。(*16)(*17)(*18) 地図は更新される。梯子の形——離散化、混合／変換、条件付け、資源、外側の検証——は、よりゆっくりしか変わらない。刺し方の本番は直前の dual playbooks にある。
